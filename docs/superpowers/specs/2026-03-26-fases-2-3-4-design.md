# EhFacturas Android — Spec Fases 2 (gaps), 3 y 4

## Resumen

Completar la app Android con paridad de features respecto a iOS:
- **Fase 2 gaps**: Formulario factura (crear/editar con líneas inline), pantalla detalle, validación de formularios
- **Fase 3**: IA on-device (Gemini Nano + Gemma 3 4B) + cloud fallback (Claude/OpenAI) + mejoras de voz
- **Fase 4**: Generación PDF A4 + sistema de hashes encadenados (preparado para VeriFactu sin envío)

**Recálculo de totales (idéntico a iOS):**

```
Para cada línea:
  subtotal = cantidad × precioUnitario × (1 - descuentoPorcentaje/100)

baseImponible = Σ subtotales
descuentoGlobal = baseImponible × descuentoGlobalPorcentaje / 100
baseConDescuento = baseImponible - descuentoGlobal

totalIVA = Σ (línea.subtotal × línea.porcentajeIVA / 100)  // respetando IVA por línea
totalIRPF = baseConDescuento × irpfPorcentaje / 100  // si negocio.aplicarIRPF
totalFactura = baseConDescuento + totalIVA - totalIRPF
```

### 2.2 FacturaDetalleScreen — Vista read-only

Pantalla read-only para facturas emitidas. Mismas secciones que FacturaEditScreen pero con Text en vez de TextField.

**Secciones adicionales:**
- **Acciones** (HorizontalScrollable row de botones):
  - PDF (genera/abre)
  - Emitir (cambia estado a EMITIDA, crea RegistroFacturacion con hash)
  - Cobrar (cambia estado a PAGADA)
  - Anular (cambia estado a ANULADA, crea registro de anulación)
  - Rectificar (crea factura rectificativa vinculada)
  - Duplicar (crea borrador con mismos datos)
- **Registros de facturación** (si existe): muestra hash, fecha generación, estado envío
- **Prompt original** (si existe): el comando de voz que generó la factura

**Navegación:**
- Desde FacturasScreen (lista) → tap → FacturaDetalleScreen
- Desde FacturaDetalleScreen → botón "Editar" (solo si estado == BORRADOR o PRESUPUESTO) → FacturaEditScreen
- Desde MainScreen (IA crea factura) → navega a FacturaEditScreen
- Acción "Convertir en factura" (si estado == PRESUPUESTO) → cambia estado a BORRADOR
- Acción "Duplicar" → crea nueva factura BORRADOR con mismos datos + nuevas fechas

**Integración con IA:**
- Cuando la IA ejecuta `crear_factura`, el ToolExecutor guarda la factura y devuelve su ID
- MainViewModel emite un evento de navegación `NavigateToFactura(facturaId)` vía SharedFlow
- MainScreen observa el evento y navega a FacturaEditScreen con el ID

### 2.3 Validación de formularios

**Validador NIF/CIF/NIE** (`domain/validation/NifValidator.kt`):
```
NIF:  ^[0-9]{8}[A-Z]$                           (12345678Z)
CIF:  ^[ABCDEFGHJKLMNPQRSUVW][0-9]{7}[0-9A-J]$  (A12345678)
NIE:  ^[XYZ][0-9]{7}[A-Z]$                       (X1234567L)
```
Validación opcional (campo puede estar vacío, pero si tiene valor debe ser válido).

**Parseo de precios** (`domain/validation/Formateadores.kt`):
```kotlin
fun parsearPrecio(texto: String): Double? {
    val limpio = texto
        .replace(",", ".")
        .replace("€", "")
        .trim()
    return limpio.toDoubleOrNull()
}
```
Acepta: "123.45", "123,45", "123,45 €"

**Validación por entidad:**

| Entidad | Campo | Regla |
|---------|-------|-------|
| Cliente | nombre | Requerido, no vacío |
| Cliente | nif | Opcional, formato NIF/CIF/NIE si presente |
| Cliente | email | Opcional, formato email si presente |
| Artículo | nombre | Requerido, no vacío |
| Artículo | precioUnitario | >= 0 |
| Factura | al menos 1 línea | Requerido para guardar |
| Factura línea | concepto | Requerido |
| Factura línea | cantidad | > 0 |
| Factura línea | precioUnitario | >= 0 |
| Gasto | concepto | Requerido, no vacío |
| Gasto | importe | >= 0 |

**Integración en UI:**
- Errores inline bajo cada campo con `supportingText` de `OutlinedTextField`
- Color `error` de Material 3
- Validación al hacer tap en "Guardar" (no en cada keystroke para evitar ruido)
- Los formularios existentes (Clientes, Artículos, Gastos) se actualizan para añadir validación

## Fase 3: IA + Voz

### 3.1 Arquitectura AI Provider — 3 niveles

```kotlin
interface AIProvider {
    val nombre: String
    val isAvailable: Boolean
    val unavailableReason: String

    suspend fun processCommand(
        texto: String,
        systemPrompt: String,
        tools: List<ToolDefinition>,
        onToolCall: suspend (String, Map<String, Any>) -> String
    ): AIResult
}
```

**Nivel 1: GeminiNanoProvider** (AI Core / ML Kit)
- Usa `GenerativeModel` de Google AI para Android
- Disponible en dispositivos con AI Core (Pixel 8+, Samsung S24+, etc.)
- Detección: `GenerativeModel.isAvailable(context)`
- Gratuito, sin API key, baja latencia
- System prompt + tool definitions vía function calling nativo de Gemini

**Nivel 2: GemmaLocalProvider** (MediaPipe LLM Inference)
- Descarga Gemma 3 4B al primer uso (~2.5GB)
- Progreso de descarga visible en UI (ProgressBar + porcentaje)
- Tool calling vía prompt engineering con instrucciones JSON + parseo manual
- Fallback a Gemma 3 1B si el dispositivo tiene < 4GB RAM libre
- Se inicializa lazy (no al arrancar la app)

**Nivel 3: CloudAIProvider** (Claude + OpenAI)
- Reusa la implementación actual (`ClaudeAIProvider`, `OpenAIProvider`)
- Se usa cuando:
  - No hay modelo local disponible
  - El modelo local falla (JSON malformado, timeout)
  - Comando complejo que el local no puede resolver
- Requiere API key configurada

**AIProviderFactory (actualizado):**
```kotlin
@Singleton
class AIProviderFactory @Inject constructor(
    private val geminiNanoProvider: GeminiNanoProvider,
    private val gemmaLocalProvider: GemmaLocalProvider,
    private val claudeProvider: ClaudeAIProvider,
    private val openAIProvider: OpenAIProvider,
    private val appPreferences: AppPreferences
) {
    suspend fun getProvider(): AIProvider {
        // 1. Gemini Nano si disponible
        if (geminiNanoProvider.isAvailable) return geminiNanoProvider

        // 2. Gemma local si descargado
        if (gemmaLocalProvider.isAvailable) return gemmaLocalProvider

        // 3. Cloud (preferencia del usuario)
        val preferred = appPreferences.cloudProvider.first()
        return when (preferred) {
            "claude" -> if (claudeProvider.isAvailable) claudeProvider else openAIProvider
            "openai" -> if (openAIProvider.isAvailable) openAIProvider else claudeProvider
            else -> claudeProvider
        }
    }

    // Para forzar cloud en comandos complejos
    fun getCloudProvider(): AIProvider { ... }
}
```

### 4.2 Hash chain — `domain/verifactu/HashService.kt`

Implementación idéntica a iOS (`VeriFactuHashService.swift`) pero sin envío a AEAT.

**Cadena de hash SHA-256:**

```
hashString = [
    nifEmisor,
    numeroFactura,
    serieFactura,
    fechaExpedicion (formato "dd-MM-yyyy"),
    tipoFactura (completa/simplificada/rectificativa),
    importeTotal (formato "%.2f"),
    hashRegistroAnterior (vacío si es el primero),
    fechaHoraGeneracion (formato "dd-MM-yyyy HH:mm:ss±HH:MM")
].joinToString("|")

hash = SHA-256(hashString).toHexString()
```

**HashService:**
```kotlin
@Singleton
class HashService @Inject constructor(
    private val registroDao: RegistroFacturacionDao
) {
    // Calcula el hash SHA-256 de un registro
    fun calcularHash(registro: RegistroFacturacion): String

    // Obtiene el hash del registro anterior (o "" si es el primero)
    suspend fun obtenerHashAnterior(): String

    // Crea registro de ALTA al emitir factura
    suspend fun crearRegistroAlta(
        factura: Factura,
        negocio: Negocio
    ): RegistroFacturacion

    // Crea registro de ANULACION al anular factura
    suspend fun crearRegistroAnulacion(
        factura: Factura,
        negocio: Negocio
    ): RegistroFacturacion

    // Verifica integridad de toda la cadena
    suspend fun verificarCadena(): CadenaVerificacion
}

data class CadenaVerificacion(
    val valida: Boolean,
    val totalRegistros: Int,
    val errores: List<String>
)
```

**Flujo de emisión de factura:**
1. Usuario pulsa "Emitir" en FacturaDetalleScreen
2. Validar que la factura está en BORRADOR y tiene líneas
3. Generar número factura (`negocio.generarNumeroFactura()`)
4. `hashService.crearRegistroAlta(factura, negocio)` → crea registro con hash encadenado
5. Cambiar `factura.estado = EMITIDA`
6. Guardar factura + registro
7. Generar PDF automáticamente
8. Navegar a FacturaDetalleScreen (ahora read-only)

**Flujo de anulación:**
1. Validar que la factura está en EMITIDA o PAGADA
2. `hashService.crearRegistroAnulacion(factura, negocio)`
3. Cambiar `factura.estado = ANULADA`
4. Guardar

**No se implementa (preparado para futuro):**
- Envío SOAP a AEAT (`estadoEnvio` queda en `NO_ENVIADO`)
- Generación XML VeriFactu (la estructura de datos está lista)
- Certificado digital
- Entorno de pruebas AEAT

- Crear factura con líneas, calcular totales correctamente (IVA por línea, IRPF, descuento global)

- PDF generado con layout correcto, compartible

- NIF/CIF/NIE validado correctamente
- Precios aceptan formato español (coma decimal)
